# Code of Conduct

## Purpose

This project is a technical collaboration space. Everyone participating in the
repository, issue tracker, code review process, discussions, and releases is
expected to keep interactions professional, respectful, and focused on the
work.

## Expected Behavior

- Be respectful, direct, and constructive.
- Critique ideas, code, and decisions, not people.
- Assume good intent until the facts show otherwise.
- Keep technical discussions evidence-based and specific.
- Document breaking changes, risks, and tradeoffs clearly.
- Help maintain a reviewable history by using clear commits and pull requests.

## Unacceptable Behavior

- Harassment, intimidation, threats, or personal attacks.
- Discrimination or exclusion based on personal characteristics.
- Repeatedly derailing technical discussions.
- Publishing private, sensitive, or security-relevant information without
  authorization.
- Misrepresenting test results, authorship, or review status.
- Submitting intentionally misleading, malicious, or destructive changes.

## Enforcement

Maintainers may remove comments, reject changes, lock discussions, or block
contributors whose behavior violates this code of conduct.

## Scope

This code of conduct applies to:

- repository discussions,
- issues and pull requests,
- code reviews,
- release and maintenance work,
- any project space where someone is representing this repository.

## Reporting

Report conduct issues privately to the maintainers through the repository
owners' preferred contact channel. Include links, timestamps, and enough
context to investigate quickly.
